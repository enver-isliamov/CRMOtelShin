# Спецификация: Синхронизация данных из UI в Google Sheets

## 1. Добавление нового клиента (Новый заказ)
- **Действие в UI:** Пользователь заполняет форму и нажимает "Сохранить".
- **Процесс в Vercel API:** 
  - Генерируется уникальный `id` (например, `c_1710672000000_1234`).
  - Клиент сохраняется в базу данных Postgres.
  - Отправляется вебхук `sync_client` в Google Apps Script (GAS).
- **Процесс в GAS:**
  - Скрипт ищет строку по `id` на листе `WebBase`.
  - Так как `id` новый, строка не находится.
  - Скрипт добавляет **новую строку** в конец листа `WebBase`.

## 2. Обновление существующего клиента
- **Действие в UI:** Пользователь редактирует данные клиента и сохраняет.
- **Процесс в Vercel API:**
  - Данные обновляются в базе Postgres по `id`.
  - Отправляется вебхук `sync_client` в GAS.
- **Процесс в GAS:**
  - Скрипт находит строку по `id` на листе `WebBase`.
  - Скрипт **обновляет существующую строку** новыми данными.

## 3. Повторный заказ существующего клиента (Reorder)
- **Действие в UI:** Пользователь нажимает "Повторный заказ" (или "Переоформить").
- **Процесс в Vercel API:**
  - Создается новый объект клиента с **новым `id`**, но старыми ФИО, телефоном и авто.
  - В базе Postgres старый заказ помечается как `is_archived = TRUE`.
  - Новый заказ сохраняется как активный.
  - **Исправление (17.03.2026):** Ранее бэкенд ошибочно отправлял два `sync_client`, что приводило к дублированию строк в `WebBase`. Теперь бэкенд отправляет специальную команду `reorder` в GAS, передавая `oldClientId` и данные нового заказа.
- **Процесс в GAS:**
  - Скрипт получает команду `reorder`.
  - Находит строку старого заказа по `oldClientId` на листе `WebBase`.
  - **Копирует** данные этой строки и переносит их на лист `Archive` (с пометкой даты архивации).
  - **Обновляет** оригинальную строку на листе `WebBase` данными нового заказа (новый `id`, договор, даты и т.д.).
  - **Результат:** Таблица остается чистой. Старый заказ ушел в архив, а на его месте появился новый активный заказ.

## 4. Удаление клиента
- **Действие в UI:** Пользователь удаляет клиента.
- **Процесс в Vercel API:**
  - Клиент удаляется из базы Postgres.
  - Отправляется вебхук `sync_delete_client` в GAS.
- **Процесс в GAS:**
  - Скрипт находит строку по `id` на листе `WebBase` и **удаляет её**.
